Design-in-Browser Mindset

I don't design in Figma. I use Canva for quick idea sketching (Rapid Prototyping) and design directly in Code (HTML/CSS/Tailwind) in the browser. This approach brings outstanding advantages for lean startups:

⚡

Cut product launch time by 50%

Skip the cumbersome hand-off step between Designer and Developer. UI/UX ideas are directly realized into working code, ready for User Testing immediately.

🧱

Mastering Components & Design System

Deep understanding of Layout structure (Flexbox, Grid), Typography, Component management through real code. Interface design based on structural reuse thinking — no junk code generated.

♿

Optimize Accessibility & Compatibility from the start

Interfaces meet SEO standards, perfect Responsive design, and follow Accessibility rules (ARIA) from the very first line of code — rather than optimizing after the design is done.

Product Thinking & UX Flow Transformation

To create high-value SaaS products for AI engineers, I always follow battle-tested design principles and accept Trade-offs in favor of user experience:

1. Translating Abstract Concepts

For tree-structured data or large-scale data computation processes, I prioritize designing linear visual workflow models (Workflow Visualizer), helping Data Scientists instantly grasp system state.

2. Testing & Iteration (User Testing & Iteration)

Instead of spending time on static mockups, I immediately build high-fidelity prototypes directly in Code (HTML/Tailwind), put the product in front of end users to collect real feedback and modify UI logic within hours.

3. Cross-team Collaboration

Thanks to a Frontend technical background, I can easily sit down with Product Managers to understand business goals, and discuss data API structure in depth with Backend Engineers, ensuring designs never drift from programming reality.
