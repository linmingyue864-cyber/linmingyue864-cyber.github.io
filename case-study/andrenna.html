// 02 — HẠ TẦNG VIỆT NAM

Máy chủ & Tên miền .vn

Toàn bộ hạ tầng ứng dụng đặt tại Việt Nam. Sử dụng tên miền .vn đăng ký qua VNNIC để đảm bảo tuân thủ pháp lý và tối ưu latency cho người dùng nội địa.

ℹ️

Tại sao phải đặt máy chủ tại Việt Nam?

Nghị định 13/2023/NĐ-CP về bảo vệ dữ liệu cá nhân yêu cầu dữ liệu của người dùng Việt Nam phải được xử lý và lưu trữ tại Việt Nam hoặc phải có cơ chế kiểm soát. Ngoài ra, latency từ IDC Hà Nội/TP.HCM xuống user nội địa đạt dưới 10ms, thấp hơn 10–20 lần so với server đặt tại Singapore hay Nhật.

🏢
IDC / Hosting Provider
Nhà cung cấp máy chủ nội địa

Sử dụng các nhà cung cấp VPS/Dedicated Server uy tín tại Việt Nam: Vietnix, AZDIGI, hoặc Viettel IDC. Data center đặt tại Hà Nội (DC1) và TP.HCM (DC2) để đảm bảo redundancy.

Vietnix VPS AZDIGI Cloud Viettel IDC 99.9% SLA
🌐
Tên miền .vn
Đăng ký qua VNNIC / PAVIETNAM

andrenna.vn — tên miền chính. Đăng ký qua PAVIETNAM hoặc VNPT. Hai subdomain chính: music.andrenna.vn và drive.andrenna.vn. DNS quản lý qua Cloudflare để tận dụng anycast routing.

andrenna.vn music.andrenna.vn drive.andrenna.vn PAVIETNAM
🔀
DNS & Proxy
Cloudflare Free Plan

Dùng Cloudflare làm DNS resolver và reverse proxy miễn phí. Kích hoạt Cloudflare Proxy (Orange Cloud) để ẩn IP thật của server, tự động cấp SSL/TLS Let's Encrypt, DDoS protection cơ bản và cache tĩnh.

Cloudflare Free SSL Auto DDoS Basic Orange Cloud
🔒
SSL / TLS
Let's Encrypt + Cloudflare

TLS 1.3 bắt buộc cho mọi endpoint. Chứng chỉ SSL được cấp tự động qua Let's Encrypt (Certbot) và renew tự động mỗi 90 ngày. Strict HSTS header đảm bảo HTTPS luôn được dùng.

TLS 1.3 Let's Encrypt HSTS Auto-renew
⚙️
Web Server
Nginx Reverse Proxy

Nginx đóng vai trò reverse proxy phía trước Node.js app. Cấu hình rate limiting, gzip/brotli compression, HTTP/2, và serve static assets trực tiếp. PM2 quản lý Node.js process với cluster mode tận dụng đa nhân CPU.

Nginx HTTP/2 PM2 Cluster Brotli
📦
Container & Deploy
Docker + GitHub Actions

Ứng dụng được đóng gói trong Docker container. Deploy tự động qua GitHub Actions CI/CD: build image → push Docker Hub → SSH vào server → pull & restart container. Zero-downtime deploy với Docker Compose rolling update.

Docker Docker Compose GitHub Actions Zero Downtime

// 03 — LƯU TRỮ & CDN

Backblaze B2 + Cloudflare CDN

Toàn bộ media (file nhạc, artwork, video, file người dùng upload) và CDN đều lưu trên Backblaze B2 — dịch vụ object storage S3-compatible với free tier hào phóng. Kết hợp với Cloudflare để serve CDN miễn phí (băng thông B2 ↔ Cloudflare là free egress theo Bandwidth Alliance).

💡

Bandwidth Alliance: Bí quyết tiết kiệm chi phí chủ chốt

Backblaze và Cloudflare là thành viên của Bandwidth Alliance — nghĩa là băng thông từ B2 đến Cloudflare CDN hoàn toàn miễn phí. Đây là lý do cốt lõi chọn B2 thay vì AWS S3 (S3 tính tiền egress 0.09$/GB). Với một dự án starter, đây là mức tiết kiệm đáng kể.

🪣

lib/storage/b2-client.ts
TypeScript
import { S3Client, PutObjectCommand, GetObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

// Backblaze B2 – S3-compatible endpoint
export const b2 = new S3Client({
  region: 'us-west-004', // B2 region của bucket
  endpoint: 'https://s3.us-west-004.backblazeb2.com',
  credentials: {
    accessKeyId:     process.env.B2_KEY_ID!,
    secretAccessKey: process.env.B2_APP_KEY!,
  },
});

export async function uploadMedia(key: string, body: Buffer, mimeType: string) {
  return b2.send(new PutObjectCommand({
    Bucket:      process.env.B2_BUCKET_NAME!,
    Key:         key,
    Body:        body,
    ContentType: mimeType,
    // Cache 7 ngày – Cloudflare sẽ cache theo header này
    CacheControl: 'public, max-age=604800, immutable',
  }));
}

export async function getPrivateUrl(key: string, expiresIn = 3600) {
  // Signed URL cho file private (Drive documents)
  return getSignedUrl(b2, new GetObjectCommand({
    Bucket: process.env.B2_BUCKET_NAME!,
    Key: key,
  }), { expiresIn });
}

// Public CDN URL (qua Cloudflare) cho media công khai như nhạc stream
export const cdnUrl = (key: string) =>
  `https://cdn.andrenna.vn/${key}`; // Cloudflare custom domain trỏ vào B2

Lưu ý về giới hạn Free Tier của B2

Free tier của Backblaze B2 bao gồm 10GB lưu trữ và 1GB egress/ngày (với non-Cloudflare). Khi dự án vượt ngưỡng này, chi phí rất cạnh tranh: $0.006/GB lưu trữ và $0.01/GB download. Với Andrenna ở giai đoạn starter, 10GB là đủ cho vài nghìn bài nhạc (MP3 128kbps ~4MB/bài = ~2500 bài).

// 05 — BẢO MẬT & TUÂN THỦ

Bảo mật đa lớp

Andrenna áp dụng mô hình bảo mật defense-in-depth: mỗi lớp độc lập, nếu một lớp bị vượt qua thì các lớp còn lại vẫn bảo vệ. Tuân thủ Nghị định 13/2023/NĐ-CP về bảo vệ dữ liệu cá nhân.

🔐
Mã hóa dữ liệu tĩnh
File trên B2 được mã hóa AES-256-GCM phía server trước khi upload. Andrenna Drive hỗ trợ tùy chọn Zero-Knowledge Encryption (ZKE) – key chỉ client giữ.
🚇
Mã hóa truyền dẫn
TLS 1.3 bắt buộc cho mọi kết nối. HSTS header với max-age 1 năm. Không có HTTP fallback. Certificate Transparency monitoring tự động.
🎫
Authentication
JWT Access Token ngắn hạn (15 phút) + Refresh Token rotation (30 ngày, HTTP-only cookie). Password hash bằng Argon2id. OAuth 2.0 PKCE flow.
🛡️
DDoS & Rate Limiting
Cloudflare absorbs DDoS tầng network. Nginx rate limiting (50 req/phút per IP cho API). Redis-based distributed rate limiter cho auth endpoints.
🔗
Signed URLs cho Media
File riêng tư chỉ accessible qua Signed URL có thời hạn (1–24h). Cloudflare Workers ký HMAC-SHA256. Không thể chia sẻ link vĩnh viễn cho file private.
🧹
Input Validation
Zod schema validation mọi input API. File type validation bằng magic bytes (không tin vào extension hay Content-Type). SQL injection prevention qua ORM parameterized queries.
📋
Tuân thủ NĐ 13/2023
Dữ liệu cá nhân người dùng Việt Nam lưu trong PostgreSQL đặt tại IDC Việt Nam. Chính sách xóa dữ liệu khi người dùng yêu cầu (Right to Erasure). Audit log mọi hành động truy cập data.
📊
Audit Log
Mọi hành động quan trọng (đăng nhập, upload, xóa, chia sẻ) được ghi vào immutable audit log lưu tách biệt. Retention 90 ngày. Không thể xóa từ application layer.
🔄
Backup & Recovery
Chiến lược backup 3-2-1: 3 bản sao, 2 loại media khác nhau, 1 bản offsite. PostgreSQL pg_dump tự động hàng ngày, upload lên B2 bucket riêng. RTO < 4h, RPO < 24h.
